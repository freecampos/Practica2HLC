Visual Studio Code para JavaScript

Introducción

Visual Studio Code se basa en Electron, un framework que se utiliza para implementar Chromium y Node.js como aplicaciones para escritorio, que se ejecuta en el motor de diseño Blink. Aunque utiliza el framework Electron, el software no usa Atom y en su lugar emplea el mismo componente editor (Monaco) utilizado en Visual Studio Team Services (anteriormente llamado Visual Studio Online).

Complementos y extensiones

Os presentamos unos cuantos de complementos y extensiones que, a nuestro juicio, merecen la pena y están ya disponibles. Algunas serán útiles para todo tipo de programadores pero otras están orientadas solamente a desarrollo Web. Algunos complementos ya lo explicamos con más profundidad en algunas de nuestras páginas, pero aqui vamos a ver y explicar algunos tambien importantes.

  • Live Server:

    • Le permite editar y depurar cualquier código base en colaboración con otros en tiempo real; con cada una de las personas trabajando desde su propio entorno familiar y personalizado.

  • Code runner:

    • Ejecute el fragmento de código o el archivo de código para varios idiomas.

  • Prettier-Code formatter:

    • Es un formateador de código opinado. Aplica un estilo coherente analizando el código y volviendo a imprimirlo con sus propias reglas que tienen en cuenta la longitud máxima de línea, ajustando el código cuando sea necesario.

Funcionamiento

Visual Studio 2019 proporciona compatibilidad enriquecida para el desarrollo con JavaScript, tanto con el uso de JavaScript directamente como con el uso del lenguaje de programación TypeScript, que se desarrolló para ofrecer una experiencia de desarrollo con JavaScript más productiva y entretenida, sobre todo al desarrollar proyectos a escala.

    Visual Studio admite ahora la sintaxis para actualizaciones del lenguaje ECMAScript

  • ¿Qué es ECMASScript ?

    • Es una actualización del lenguaje JavaScript que incorpora funciones y sintaxis nuevas de gran utilidad.

Transpilación de JavaScript

Un problema común con JavaScript es que querrá usar las características de lenguaje ES6+ más recientes porque le ayudan a ser más productivo, pero sus entornos en tiempo de ejecución (con frecuencia exploradores) no admiten aún estas nuevas características. Una de las principales características de TypeScript es la posibilidad de transpilar código,de forma que pueda escribir el código que le permita ser más productivo, pero seguir ejecutando el código en cualquier plataforma. Como JavaScript en Visual Studio 2017 usa el mismo servicio de lenguaje que TypeScript, también puede aprovechar ES6+ para la transpilación de ES5.
